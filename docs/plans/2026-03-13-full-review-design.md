## A. 全局 UI/UX 规范

### A1. 组件库统一 ✅

**决定**：移除 ALL Ant Design 依赖，全面迁移到 shadcn/ui + 自定义 Tailwind 组件。

- Products / Iterations 页面必须按原型（`Sisyphus-Y.html`）完全重建
- 安装 **Sonner** 作为全局 Toast 系统：
  - `<Toaster position="top-right" theme="dark" />`
  - 删除类 Toast 带 **Undo 撤销按钮**，`duration=5000ms`
  - 异步操作使用 `toast.promise` 自动切换 loading → success / error
- 迁移完成后 `package.json` 中 **零 antd 相关依赖**

### A2. 颜色 Token 统一 ✅

**决定**：全局替换所有硬编码颜色值（`rgba()`、`style={{ color: '#xxx' }}`、`text-[#xxx]`）为 `sy-*` token + opacity 修饰符。

### A3. Loading 动画统一 ✅

**决定**：分场景处理：

| 场景 | 方案 |
|------|------|
| 列表/表格首次加载 | 骨架屏 Skeleton（随机宽度占位行） |
| 按钮触发操作 | `<Loader2 className="animate-spin" />` + 文字变化（「保存中...」），按钮禁用防重复 |
| 页面路由跳转 | 顶部细线进度条（nprogress 方案） |
| 弹窗内容加载 | Spinner 居中，24px，无文字 |

**禁止**：`"加载中..."` 三个字单独作为页面主体内容。

### A4. 错误处理统一 ✅

**决定**：

- **500 错误**：必须展示 response body 中的具体错误消息，不允许只显示"操作失败"
- **网络错误**：区分 timeout（「请求超时，请检查网络连接后重试」）和 offline（「网络连接已断开」）
- 所有 API 错误通过 Sonner Toast 展示，`console.log` 仅用于调试

### A5. 表格组件统一 ✅

**决定**：开发自定义 `DataTable` 组件，统一所有列表页的表格样式和行为。

### A6. 表单方案统一 ✅

**决定**：**React Hook Form + Zod + shadcn/ui Form** 集成方案。所有表单统一使用此技术栈，禁止手动 `useState` 管理表单状态。

## B. 导航菜单与页面重构

### B1. 菜单结构 ✅

从 13 个菜单项精简为 **9 个**，按分隔线分为 4 组：

```
1. 仪表盘      → 项目概览 Tab + 质量分析 Tab
── 分隔线 ──
2. 分析台      → 需求列表 + AI分析 + 覆盖追踪
3. 工作台      → Step1 确认测试点 + Step2 生成用例
4. 需求Diff    → 版本对比 + 变更影响
── 分隔线 ──
5. 用例库      → 目录树 + 用例列表 + 导入/导出
6. 模板库      → Prompt模板 + 用例结构模板
7. 知识库      → 四分类文档管理 + 手动条目
── 分隔线 ──
8. 回收站      → 30天自动清理
9. 设置        → AI配置(模型列表+Prompt管理) + 操作日志
```

### B2. 术语变更 ✅

**「诊断」全局替换为「分析」**，覆盖范围包括：前端菜单、页面标题、按钮文字、后端日志输出、数据库字段注释。

### B3. 分析台页面设计 ✅

**布局**：左右两栏

- **左栏**：需求列表，按迭代分组，每条需求显示状态 Badge（`未分析` / `分析中` / `已完成`）
- **右栏**：3 个 Tab
  - **需求详情**：需求内容编辑/查看，编辑后出现「开始分析」按钮
  - **AI 分析**：顶部展示广度扫描结果 + 底部苏格拉底追问对话（同一 Tab 内）
  - **覆盖追踪**：需求 × 场景类型的覆盖矩阵

**交互规则**：
- 点击「开始分析」后自动切换到 AI 分析 Tab
- 未解决的高风险遗漏项 → 「进入工作台」按钮禁用，hover 显示 Tooltip 说明原因

### B4. 工作台页面设计 ✅

**布局**：顶部 Step 进度条 + 三栏内容区，支持双向切换

**Step 1 — 确认测试点**：
- 左栏：需求导航树
- 中栏：AI 生成的测试点列表，按场景类型分组，每个测试点可勾选/取消
- 右栏：已选测试点汇总 + RAG 相似历史用例推荐

**Step 2 — 生成用例**：
- 左栏：需求树 + 已确认测试点（只读）
- 中栏：SSE 流式对话区域
- 右栏：生成的用例卡片，按测试点分组

可从 Step 2 返回 Step 1 追加测试点。

## D. System Prompt 标准化

### D1. 6 个模块全部重写 ✅

所有模块 System Prompt 统一 4 段结构：

```
① 身份声明（角色 + 专业背景，每个模块不同）
② 任务边界（只做什么，不做什么）
③ 输出规范（格式/语言/禁止项）
④ 质量红线（不合格输出标准，必须重试）
```

**身份声明示例**（每个模块必须差异化）：
- 分析模块：「你是 Sisyphus-Y 平台的需求质量分析专家，拥有 10 年数据中台测试经验...」
- 用例生成：「你是 Sisyphus-Y 平台的资深测试工程师，专注为数据中台系统编写可直接执行的测试用例...」
- Diff 分析：「你是 Sisyphus-Y 平台的测试影响分析专家，专注评估需求变更对已有用例的影响范围...」

重写时参考 `docs/PROMPT_RULES.md` 现有模板，不从零开始。

### D2. Prompt 管理 UI ✅

在设置页「AI 配置 → Prompt 管理」新增 Tab：

- 6 个模块各一个编辑入口，展示当前 System Prompt
- 编辑器：多行文本，等宽字体，字数统计
- 「重置为默认值」按钮，一键还原
- 保存即生效，无需重启
- 页面顶部全局提示：「修改 System Prompt 会直接影响 AI 输出质量，建议在测试环境验证后再用于生产」
- 保存历史：每次保存记录上一版本，支持查看最近 **5 次修改记录并回滚**

## E. 用例生成质量提升

### E1. 历史用例质量审查 ✅

**流程**：启动并行 subagent → 调用 `/functional-test-case-reviewer` skill → 逐一审查 `清洗后数据/` 目录。

**审查标准**：
1. 第一步必须是「进入 [完整页面路径] 页面」
2. 每个步骤有对应预期结果，多个预期用 `1) 2) 3)` 格式
3. 前置条件涉及数据库的必须有完整建表 + 插入语句
4. 步骤完全独立，不依赖其他用例
5. 预期结果具体可验证，禁止「操作成功」「显示正确」「无报错」

**处置规则**：
- ✅ 通过 → 直接入向量库
- 🔧 可修复 → subagent 自动润色 → 入库
- ❌ 不可修复 → 丢弃并记录原因

**输出**：审查报告（总数 / 通过数 / 润色数 / 丢弃数 / 丢弃原因汇总）

### E2. RAG 检索优化 ✅

- Top-K = **5**
- 相似度阈值 = **0.72**，低于阈值不强凑
- 注入 Prompt 时带相似度分数

### E3. Prompt Few-shot 增强 ✅

在现有 7 层 Prompt 结构上补充：

- **正例**（3 条）：从审查通过的高质量用例中选取，覆盖正常/异常/边界场景
- **负例**（3 条）：典型低质量（模糊预期/步骤不独立/前置条件缺失），标注「禁止生成此类内容」

Few-shot 示例从审查通过的真实数据中选取，不手写。

### E4. GLM-5 支持 ✅

模型 ID：`glm-5`，Base URL 不变。

### E5. 历史数据清洗修复 ✅

**执行顺序**：
1. 先修复前端渲染（`\n` → 换行显示）
2. 检查 DB 原始数据（抽样 10 条）：若 DB 干净 → 仅渲染问题；若 DB 脏 → 重跑清洗
3. 修复字段名中英文混杂（统一为中文：前置条件、步骤、预期结果等）
4. 合并到 E1 审查流程中一次性完成
5. **清空旧向量记录后重新入库**（避免重复向量）

## F. 文档上传与预处理

### F1. 标准模板下载 ✅

分析台需求录入区提供「下载模板」按钮，两种格式：
- `需求文档模板.docx`
- `需求文档模板.md`

模板内置结构：
```markdown
# 需求标题
## 功能背景
## 功能描述
### 功能点 1：xxx
- 业务规则：
- 异常处理：
- 数据约束：
### 功能点 2：xxx
## 接口说明（可选）
## 非功能要求（可选）
```

上传入口旁说明文字：「建议使用标准模板上传，结构化识别效果更佳」

模板文件放 `public/templates/`，前端静态下载，不走后端接口。

### F2. 上传后自动结构化拆分 ✅

UDA 解析完成后，按章节/功能点拆分为独立需求条目 → 展示在左侧需求列表。

用户可 **确认 / 合并 / 拆分 / 删除** 条目，确认后才正式保存。

非标准文档：提示「文档结构识别置信度较低，建议对照模板调整后重新上传」，不报错、不阻断。

## G. 用例库增强

### G1. 自定义目录管理 ✅

**目录结构**（自动生成顶层）：

```
产品名称（如: 离线开发）
└── 迭代版本（如: v6.4.6）
    ├── xxx需求1
    ├── xxx需求2
    └── xxx需求3
```

**规则**：

| 功能 | 规则 |
|------|------|
| 嵌套层级 | 最大 3 层，超出置灰 + hover 提示 |
| 重命名 | 双击编辑，回车确认，Esc 取消，≤20 字，不可为空，不可同级重名 |
| 删除空目录 | 直接删除，无弹窗 |
| 删除非空目录 | 二次确认：「该目录下有 N 条用例，删除后用例将移入回收站，目录结构不可恢复，是否继续？」 |
| 拖拽排序 | 同级目录间拖拽，持久化。不支持跨层级拖拽 |
| 跨层级移动 | 右键菜单「移动到...」 |
| 拖拽用例 | 单条/多选批量拖拽到目标目录（高亮反馈） + 右键「移动到...」 |
| 未分类目录 | 不可删除/重命名，未指定目录的用例自动归入 |
| 批量编辑 | 支持批量修改用例所属目录 |

### G2. 用例导入 ✅

**支持格式**：Excel (.xlsx) / CSV / JSON / XMind

**XMind 解析**：使用 `xmindparser` 库（`pip install xmindparser`），支持 xmind2026 版本。

**导入流程**：
1. 选择文件（旁边有「下载导入模板」按钮，提供 Excel/CSV/XMind 三种模板）
2. **标准模板** → 自动识别 → 预览步骤
3. **非标准文件** → 字段映射步骤（左列名 → 右用例字段下拉），必填字段未映射则「确认导入」置灰
4. **预览步骤**：展示前 5 条解析结果，确认无误再导入
5. **目标目录**：弹窗中选择（复用 FolderTree），默认选中「未分类」
6. **重复检测**（标题 + 目录）：在预览步骤单独列出重复条目，逐条/批量选择：
   - 覆盖（替换现有）
   - 跳过（保留现有）
   - 重命名导入（标题加 `_imported` 后缀）
7. 全部处理完才能点「确认导入」
8. 完成 Toast：「成功导入 N 条，跳过 M 条，覆盖 K 条」

### G3. 用例导出 ✅

**支持格式**：Excel (.xlsx) / CSV / XMind / Markdown

**导出范围**（弹窗中选择）：
- 当前目录（含子目录）
- 按需求筛选
- 按迭代筛选
- 自由勾选（多选后「导出已选」）

**自定义字段**（复选框选择导出列）：

| 字段 | 默认勾选 |
|------|----------|
| 用例编号 | ✅ |
| 用例标题 | ✅ |
| 优先级 | ✅ |
| 场景类型 | ✅ |
| 前置条件 | ✅ |
| 步骤 | ✅ |
| 预期结果 | ✅ |
| 所属需求 | ✅ |
| 所属迭代 | ✅ |
| 质量分数 | ❌ |
| 来源 | ❌ |
| 创建时间 | ❌ |

XMind 格式不支持自定义字段，固定全量导出。XMind 结构：`迭代 → 测试点 → 用例 → (前置条件, 步骤+预期)`。

Excel 样式：深色表头背景、字段名加粗、步骤列自动换行、列宽自适应。

## H. 模块细节设计

### H1. 知识库管理 ✅

**4 个固定分类**（不允许用户自定义）：

| 分类 | 内容 |
|------|------|
| 企业测试规范 | 团队内部测试标准、流程文档 |
| 业务领域知识 | 产品说明书、业务规则、数据字典 |
| 历史用例 | 清洗后数据（只读，由清洗流程自动导入） |
| 技术参考 | 接口文档、数据库设计、技术方案 |

**分块预览**：文档卡片「查看分块」按钮 → 抽屉列出所有 chunk（序号/token 数/内容）。支持手动删除单个 chunk。

**手动知识条目**：「新建知识条目」→ 标题/分类/内容/标签 → 保存后向量化入库。列表中用「手动」Badge 标注。

**版本管理**：
- 同名文档上传 → 确认「是否作为新版本上传」
- 保留最近 **3 个版本**，旧版本停用检索（不删除）
- 文档卡片显示版本号，可查看历史并重新激活旧版本
- 超过 3 个版本 → 最旧版本自动物理删除

RAG 检索支持指定分类或全库检索。

### H2. 需求 Diff ✅

**触发方式**：用户在需求详情页点击「发布新版本」按钮 → 弹窗填版本说明（选填）→ 自动触发 Diff 分析。

**结果展示**（左右分栏 + Tab 切换）：
- **文本对比** Tab：代码 Diff 风格，左旧右新，删除红/新增绿/修改黄，行号对齐，折叠未变更段落
- **变更摘要** Tab：卡片列表，每卡片显示变更类型 + 内容摘要 + AI 业务影响说明

**影响分析**（自动执行）：
- 受影响用例自动标记：`需重写` / `需复核` / `不受影响`
- 结果页底部展示受影响用例列表（标题 + 影响原因 + 处理状态）
- 「一键推送到工作台」按钮 → 将「需重写」用例批量推送重新生成
- 用例列表同步显示对应 Badge

### H3. 回收站 ✅

- 保留 **30 天**自动清理，每条显示「还剩 X 天到期」，到期前 3 天标红
- 支持手动清空（二次确认：「回收站中的数据删除后不可恢复，确认清空？」）
- 顶部 Tab 筛选：全部 / 需求 / 用例 / 知识库文档
- 目录删除后其下用例拆散显示在「用例」Tab，带原目录路径
- 恢复逻辑：原目录存在 → 恢复原位；不存在 → 提示后恢复到「未分类」
- 批量恢复：统一处理，完成后 Toast 汇总

### H4. 模板库 ✅

**两个 Tab**：

**Prompt 模板**：
- 字段：模板名称 / 适用模块（分析/用例生成/Diff）/ System Prompt 内容 / 备注
- 使用时在对应模块一键加载，覆盖当前配置

**用例结构模板**：
- 字段：模板名称 / 适用场景类型 / 前置条件模板 / 步骤模板（含占位符 `{{功能名称}}`）/ 预期结果模板
- 生成用例时可选择套用，AI 在此基础上填充
